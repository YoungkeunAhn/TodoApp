# 통계 기능 (Statistics)

> [← 메인 문서로 돌아가기](./README.md) · 관련: [데이터 구조](./data-model.md) · [할 일 기능](./todo.md)

카테고리별로 두 가지 숫자를 보여준다: **남은 할 일 개수**와 **평균 달성속도**.

---

## 1. 카테고리별 남은 할 일 개수

**정의**
- 해당 카테고리에서 아직 **완료하지 않은(`is_done = false`)** 할 일의 개수.

**계산 방법**
```
남은 개수 = 그 카테고리의 할 일 중 is_done == false 인 것의 수
```

**예시**
| 카테고리 | 남은 할 일 개수 |
| --- | --- |
| 공부 | 3 |
| 운동 | 1 |

## 2. 카테고리별 평균 달성속도

**정의**
- "제때 끝냈는지, 늦게 끝냈는지"를 나타내는 값.
- 한 할 일의 달성속도 = **`done_date` − `due_date`** (일 단위).

**값의 의미**
| 값 | 의미 |
| --- | --- |
| 음수 | 마감보다 **일찍** 완료 (예: -2일 → 이틀 먼저 끝냄) |
| 0 | 마감 **당일** 완료 |
| 양수 | 마감보다 **늦게** 완료 (예: +3일 → 사흘 늦게 끝냄) |

**계산 방법**
```
1) 그 카테고리에서 완료된(is_done == true) 할 일만 모은다.
2) 각 할 일마다 (done_date − due_date) 를 일 수로 계산한다.
3) 그 값들의 평균을 낸다.

평균 달성속도 = (각 할 일의 (done_date − due_date) 합) ÷ (완료된 할 일 개수)
```

**예시 (공부 카테고리)**
| 할 일 | due_date | done_date | 달성속도(일) |
| --- | --- | --- | --- |
| 수학 문제집 | 2026-06-23 | 2026-06-24 | +1 |
| 독후감 쓰기 | 2026-06-20 | 2026-06-19 | -1 |

```
평균 달성속도 = (+1 + -1) ÷ 2 = 0일  → 평균적으로 마감일에 맞춰 끝냄
```

**규칙**
- 완료된 할 일이 하나도 없으면 평균을 계산할 수 없으므로 "기록 없음"으로 표시한다.

## 3. 화면 표시

- 통계 표의 카테고리 칸에는 이름 앞에 **색 점(dot)** 이 붙어, 사이드바·할 일 목록과
  같은 색으로 카테고리를 한눈에 구분할 수 있다 — [카테고리 색상](./category.md#5-카테고리-색상) · [ADR 0010](./adr/0010-category-color.md).

## 4. 화면에서 확인할 결과

- [ ] 카테고리마다 남은 할 일 개수가 정확히 표시된다.
- [ ] 할 일을 완료하면 그 카테고리의 남은 개수가 1 줄어든다.
- [ ] 카테고리마다 평균 달성속도(일)가 표시된다.
- [ ] 완료 기록이 없는 카테고리는 "기록 없음"으로 표시된다.
- [ ] 카테고리 이름 앞 색 점이 사이드바·할 일 목록과 같은 색으로 보인다.
