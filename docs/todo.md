# 할 일 기능 (Todo)

> [← 메인 문서로 돌아가기](./README.md) · 관련: [데이터 구조](./data-model.md) · [날짜별 화면 규칙](./date-view.md)

할 일은 이 앱의 핵심 데이터다. **추가 / 조회 / 완료 체크 / 삭제** 네 가지 동작을 지원한다.

---

## 1. 할 일 추가

**입력값**
- `title` (필수): 할 일 제목
- `category_id` (필수): 어떤 카테고리에 속하는지 선택
- `due_date` (필수): 해야 하는 날짜

**동작 순서**
```
1) 제목 입력 + 카테고리 선택 + 날짜 선택
2) [추가] 클릭
3) is_done = false, done_date = (없음) 으로 저장
4) 할 일 목록을 다시 불러와 화면(밀린/오늘)에 반영
```

**규칙**
- 제목·카테고리·날짜 중 하나라도 비면 추가하지 않는다.

## 2. 할 일 조회

**무엇을 하나**
- 저장된 할 일을 불러와 화면에 보여준다.
- [날짜별 화면 규칙](./date-view.md)에 따라 **밀린 할 일 / 오늘 할 일**로 나눠 표시한다.

**동작 순서**
```
1) Supabase todos 테이블에서 조회 (카테고리 정보와 함께)
2) 오늘 날짜를 기준으로 밀린/오늘로 분류
3) 각 영역에 표시
```

## 3. 완료 체크

**무엇을 하나**
- 할 일 옆 체크박스를 누르면 완료 처리한다.

**동작 순서**
```
1) 체크박스 클릭
2) is_done = true 로 변경
3) done_date = 오늘 날짜 로 기록
4) 목록을 다시 불러와 화면에서 빠지게 함
```

**규칙**
- 완료한 할 일은 밀린/오늘 목록에서 더 이상 보이지 않는다.
- (선택) 체크를 해제하면 is_done = false, done_date = (없음) 으로 되돌린다.
- 완료한 날짜는 [통계](./statistics.md)의 **평균 달성속도** 계산에 쓰인다.

## 4. 할 일 삭제

**동작 순서**
```
1) 할 일 옆 [삭제] 클릭
2) Supabase todos 테이블에서 해당 행 삭제
3) 목록을 다시 불러와 화면에서 제거
```

## 5. 화면에서 확인할 결과

- [ ] 할 일을 추가하면 날짜에 맞는 영역에 나타난다.
- [ ] 완료 체크를 하면 목록에서 사라지고, 완료한 날짜가 기록된다.
- [ ] 삭제하면 목록에서 사라지고, 새로고침해도 다시 나타나지 않는다.
