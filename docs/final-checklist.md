# 완성 후 검증 체크리스트 (실습용)

> [← 메인 문서로 돌아가기](./README.md)
>
> 실제로 배포/실행한 웹앱을 화면에서 직접 눌러 보며 확인한다.
> Supabase 대시보드의 **Table Editor** 도 함께 띄워 놓고 비교하면 좋다.

현재 구현 파일: `index.html`, `style.css`, `config.js`, `app.js`
DB 파일: `docs/schema.sql`, `docs/category_stats_view.sql`

---

## 0. 준비 확인

- [ ] `config.js`에 본인 **Project URL**과 **anon public key**가 들어가 있다.
- [ ] `service_role` 키는 코드 어디에도 들어가 있지 **않다**.
- [ ] Supabase에서 `schema.sql` → `category_stats_view.sql` 순서로 실행했다.

## 1. 카테고리

- [ ] 카테고리 이름을 입력하고 [추가]를 누르면 **칩 목록에 바로** 나타난다.
- [ ] 할 일 추가 영역의 **카테고리 선택 메뉴**에도 그 카테고리가 보인다.
- [ ] Supabase Table Editor의 `categories` 테이블에도 같은 행이 생겼다.
- [ ] 카테고리 항목에 hover 하면 **✎(수정) / ×(삭제)** 버튼이 보인다.
- [ ] ✎로 이름을 바꾸면 목록·할 일·통계의 카테고리 이름이 **함께** 갱신된다.
- [ ] 이미 있는 이름으로 수정하면 **중복 안내**가 뜨고 바뀌지 않는다.
- [ ] ×로 삭제하면 확인창 뒤 목록에서 사라지고 `categories` 테이블에서도 빠진다.
- [ ] **할 일이 연결된** 카테고리를 삭제하면 "할 일 N개도 함께 삭제됩니다" 경고가 뜨고,
      확인 시 `todos` 테이블의 해당 할 일도 함께 사라진다(ON DELETE CASCADE).

## 2. 할 일 추가 / 조회

- [ ] 제목 + 카테고리 + 날짜를 넣고 [추가]하면 목록에 나타난다.
- [ ] 목록의 각 할 일에 **카테고리 이름**이 함께 보인다. (categories JOIN 확인)
- [ ] `todos` 테이블에도 행이 생기고 `category_id`가 올바르게 연결돼 있다.

## 3. 완료 체크 (checked / completed_at)

- [ ] 체크박스를 **켜면** Table Editor에서 `checked = true`, `completed_at = 오늘 날짜`가 된다.
- [ ] 체크박스를 **끄면** `checked = false`, `completed_at = (비어 있음/null)`로 돌아간다.
- [ ] 완료한 할 일은 제목에 취소선이 표시된다.

## 4. 삭제

- [ ] [삭제]를 누르면 화면 목록에서 사라진다.
- [ ] Supabase `todos` 테이블에서도 해당 행이 사라진다. (양쪽 모두 삭제 확인)

## 5. 저장 / 새로고침

- [ ] 데이터가 실제로 **Supabase DB**에 저장된다. (Table Editor로 확인)
- [ ] 브라우저를 **새로고침**해도 카테고리·할 일이 그대로 다시 보인다.

## 6. 날짜별 화면

- [ ] **밀린 할 일**: 미완료이고 `target_date`가 기준 날짜보다 이전인 것이 보인다.
- [ ] **오늘 할 일**: `target_date`가 기준 날짜와 같은 것이 보인다.
- [ ] 완료하지 않은 할 일은 날짜가 지나도 **계속 보인다.** (며칠 밀렸는지도 표시)
- [ ] 밀린 할 일의 **원래 날짜(target_date)는 바뀌지 않는다.**

## 7. 날짜 이동

- [ ] [이전]/[다음]을 누르면 기준 날짜가 하루씩 바뀐다.
- [ ] [오늘]을 누르면 오늘 날짜로 돌아온다.
- [ ] 날짜 선택(달력)으로 직접 고른 날짜가 화면에 반영된다.
- [ ] 날짜를 바꾸면 밀린/오늘 목록이 그 기준에 맞게 다시 그려진다.

## 8. 카테고리별 통계 (category_stats View)

- [ ] 카테고리별 **남은 개수**가 실제 미완료 할 일 수와 일치한다.
- [ ] 할 일을 완료/삭제하면 통계가 자동으로 갱신된다.
- [ ] **평균 달성속도**가 (완료한 날짜 − 해야 했던 날짜)의 평균과 맞다.
      - 음수=일찍, 0=당일, 양수=늦게
- [ ] 완료 기록이 없는 카테고리는 **"기록 없음"** 으로 표시된다.

## 9. 입력값 검증

- [ ] 카테고리 이름을 비워 두고 [추가]하면 추가되지 않는다.
- [ ] 할 일에서 제목/카테고리/날짜 중 하나라도 비면 **안내 문구**가 뜬다.

## 10. 모바일 화면

- [ ] 브라우저 폭을 좁히거나 휴대폰으로 열어도 레이아웃이 깨지지 않는다.
- [ ] 입력칸/버튼이 세로로 정렬되어 누르기 편하다.
- [ ] 통계 표가 화면을 벗어나지 않는다.
