# ADR 0010 — 카테고리 색상(색 점으로 구분)

- 상태: **채택됨**
- 관련: [카테고리 기능](../category.md) · [데이터 구조](../data-model.md) · [add-category-color.sql](../add-category-color.sql)

## 맥락 (Context)

- 카테고리가 늘어나면 사이드바·통계·할 일 목록·추가 드롭다운에서
  어느 항목이 어떤 카테고리인지 한눈에 구분하기 어렵다.
- 카테고리마다 색을 주면 구분이 쉬워진다. 단, 색을 **어떻게 표현할지**가 문제다.
  - 글자색/배경색으로 칠하면: 가독성(명암 대비)이 깨지고, 라이트/다크 테마에서
    색마다 따로 대비를 맞춰야 하며, "색맹/저시력"에게 정보가 색에만 실린다.
- 색을 **무엇을 기준으로** 정할지도 선택지였다.
  - (a) 순서 기반(현재 사이드바의 `nth-child`) → 정렬/추가로 색이 바뀜(불안정).
  - (b) 이름 해시 → 안정적이지만 사용자가 못 고름.
  - (c) DB에 저장 + 사용자가 선택 → 안정적이고 자유로움.

## 결정 (Decision)

- 색은 **작은 색 점(dot)** 으로만 표현한다. 글자색·배경색은 바꾸지 않는다.
  - 점은 이름 옆 보조 신호라서 명암 대비/테마 문제를 일으키지 않는다.
  - 이미 노션 스타일 사이드바가 쓰던 "컬러 도트" 언어를 그대로 확장한다.
- 색은 **카테고리별로 DB에 저장**한다. `categories.color`에 **색 키**
  (`blue/green/orange/red/purple/yellow/pink/gray`)를 넣는다.
  - 키 → 실제 hex 매핑은 프론트의 `CATEGORY_COLORS` 팔레트가 단일 소스다.
  - `color`가 비어 있으면(예전 데이터) 프론트가 **이름 해시로 기본색**을 준다.
- 색은 **사용자가 바꿀 수 있다**. 사이드바 카테고리의 **색 점을 누르면**
  팔레트 팝오버가 떠서 8색 중 고른다(사이드바가 색의 "주인").
- 색 점을 **모든 화면에 일관 적용**한다: 사이드바 · 통계 표 · 할 일 태그 ·
  할 일 추가 드롭다운. 같은 카테고리는 어디서나 같은 색이다.
- 할 일 추가의 카테고리 선택은 네이티브 `<select>`가 항목 안에 점을 넣지 못하므로
  **커스텀 드롭다운**으로 교체한다(점 + 이름 표시).

## 결과 (Consequences)

**좋은 점**
- 글자/배경 대비를 해치지 않으면서 카테고리 구분이 쉬워진다.
- 정렬·추가·이름변경에도 카테고리 색이 안정적으로 유지된다(순서 기반의 불안정 제거).
- 색은 보조 신호이고 **이름 텍스트가 항상 함께** 있어, 색에만 의존하지 않는다(접근성).

**감수하는 점**
- `categories`에 컬럼이 하나 늘고, 기존 DB에는 마이그레이션 SQL을 1번 실행해야 한다.
- 네이티브 select를 커스텀 드롭다운으로 바꾸면서, 열고/닫고/바깥 클릭/ESC 같은
  동작을 직접 구현해야 한다(코드가 약간 늘어난다).
- 팔레트는 8색 고정이다. 더 많은 색이 필요하면 팔레트를 확장해야 한다.
