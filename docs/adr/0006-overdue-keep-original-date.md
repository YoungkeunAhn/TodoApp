# ADR 0006 — 밀린 할 일은 원래 날짜를 유지

- 상태: **채택됨**
- 관련: [날짜별 화면 규칙](../date-view.md)

## 맥락 (Context)

- 완료하지 않은 할 일은 **마감일이 지나도 사라지면 안 된다**(계속 보여야 함).
- "밀린 일"을 처리하는 흔한 방법 두 가지:
  1. 지난 할 일의 `target_date`를 오늘로 자동으로 옮긴다.
  2. 날짜는 그대로 두고, 화면에서 "밀림"으로 분류만 한다.

## 결정 (Decision)

- 할 일의 `target_date`(원래 해야 했던 날짜)는 **절대 자동으로 바꾸지 않는다.**
- 화면 분류 규칙(기준 날짜 대비):
  - **밀린 할 일** = 미완료 + `target_date < 기준 날짜`
  - **오늘 할 일** = `target_date == 기준 날짜`
- 밀린 항목에는 "며칠 밀렸는지"를 계산해 함께 보여준다.

## 결과 (Consequences)

**좋은 점**
- 원래 마감일이 보존되어 **평균 달성속도**(완료일 − 마감일) 계산이 정확해진다.
- 얼마나 밀렸는지 사용자가 인지할 수 있다.

**감수하는 점**
- "밀림/오늘" 분류 로직을 프론트에서 기준 날짜 기반으로 계산해야 한다.
- 날짜 비교 시 시간대(UTC↔로컬) 처리에 주의해야 한다
  (실제로 toISOString 시차 버그가 있었고, 로컬 포맷 함수로 수정함).
