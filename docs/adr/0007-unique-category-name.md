# ADR 0007 — 카테고리 이름 중복 방지(UNIQUE)

- 상태: **채택됨**
- 관련: [카테고리 기능](../category.md), [fix-category-unique.sql](../fix-category-unique.sql)

## 맥락 (Context)

- 같은 이름의 카테고리("공부")가 여러 번 저장되는 문제가 발견됐다.
- 중복 카테고리는 통계(카테고리별 집계)를 나누어 망가뜨리고 사용자를 혼란스럽게 한다.
- 막을 수 있는 위치는 두 곳: **프론트엔드 검사**와 **DB 제약**.

## 결정 (Decision)

- **두 군데 모두** 막는다.
  - 프론트: 추가 전 같은 이름(대소문자·앞뒤 공백 무시)이 있으면 안내하고 중단.
  - DB: `categories.name`에 **UNIQUE 제약**을 건다(동시 추가까지 차단).
- DB UNIQUE 위반(코드 `23505`)이 나면 프론트에서 "이미 있습니다" 안내로 변환한다.
- 이미 중복이 쌓인 기존 DB는 마이그레이션 SQL로 정리 후 제약을 적용한다.

## 결과 (Consequences)

**좋은 점**
- 어떤 경로(프론트 우회, 동시 요청)로도 중복이 생기지 않는다.
- 통계가 카테고리당 한 줄로 정확히 집계된다.

**감수하는 점**
- 대소문자·공백 정규화 기준을 프론트와 DB가 다르게 볼 수 있다
  (현재 DB 제약은 정확히 같은 문자열 기준 — 필요 시 정규화 컬럼 도입 고려).
- 기존 중복 데이터는 별도 마이그레이션이 필요하다.
