# ADR 0008 — UI를 Notion 스타일 + 라이트/다크 테마로 개편

- 상태: **채택됨**
- 관련: [0002 정적 프론트엔드](./0002-vanilla-frontend.md) (디자인 방식 갱신)

## 맥락 (Context)

- 초기 UI는 보라색 Material(MUI/Figma 참고) **카드 스택**이었는데, "사이드 프로젝트" 느낌이 강해 실제 운영 서비스처럼 보이지 않았다.
- 운영 가능한 수준의 정돈된 SaaS 느낌이 필요했고, **라이트/다크** 모두 지원하길 원했다.
- 단, 백엔드/데이터 로직(`app.js`)은 그대로 두고 **보기만** 바꾸는 게 안전하다(기능 회귀 방지).

## 결정 (Decision)

- 레퍼런스를 **Notion 스타일**로 잡고 화면을 전면 개편한다.
  - 카드 스택 → **좌측 사이드바 앱 셸**(워크스페이스 헤더 · 네비게이션 · 카테고리 · 테마 토글) + 본문(브레드크럼 · 페이지 타이틀 · 인라인 컴포저 · 블록 섹션).
  - **시스템 폰트** 스택(Segoe UI / Malgun Gothic 등), 워밍 뉴트럴 팔레트(텍스트 `#37352f`), **단일 액센트(블루)**, 그림자 대신 **얇은 경계선 + 은은한 hover**, 작은 라운드.
  - 카테고리·밀림은 Notion 셀렉트 **태그**, 섹션은 **컬러 도트** 헤더, 체크박스는 폰트 의존 없는 **인라인 SVG** 체크.
- **라이트/다크 테마**는 `:root` / `[data-theme]` **CSS 변수**로 토큰화하고, 사이드바 하단 토글로 전환한다.
  - 선택값은 `localStorage`에 저장하고, 첫 방문은 `prefers-color-scheme`를 따른다.
  - 페인트 전 `<head>` 인라인 스크립트로 테마를 먼저 적용해 **깜빡임을 방지**한다.
- **모바일**: 사이드바를 `position: fixed`로 숨기고 햄버거로 슬라이드한다. 열렸을 때 **백드롭 탭** 또는 **항목 선택**으로 닫는다.
- **`app.js`(데이터 로직)는 변경하지 않는다.** 로직이 의존하는 DOM id/클래스를 그대로 유지하고, 테마 토글·모바일 메뉴는 `index.html`의 **UI 전용 스크립트**로 분리한다.

## 결과 (Consequences)

**좋은 점**
- 실제 SaaS 제품에 가까운 정돈된 화면과 **다크 모드**를 함께 얻는다.
- 색·간격을 CSS 변수로 토큰화해 테마 변경/유지보수가 쉽다.
- 로직과 분리돼 **기능 회귀 없이** 디자인만 교체할 수 있었다.

**감수하는 점**
- 정적(프레임워크 없음) 구조라 테마 토글·반응형 메뉴를 직접 작은 스크립트로 처리한다.
- 사이드바 z-index·백드롭 등 레이아웃 상호작용을 직접 관리해야 한다
  (열린 사이드바가 햄버거를 덮어 못 닫던 버그를 백드롭/항목 선택 닫기로 해결한 이력 있음).
- Material 디자인을 다루던 [ADR 0002](./0002-vanilla-frontend.md)의 디자인 서술은 더 이상 유효하지 않다(본 ADR로 갱신).
