# 날짜별 화면 규칙 (Date View)

> [← 메인 문서로 돌아가기](./README.md) · 관련: [할 일 기능](./todo.md)

할 일을 **밀린 할 일**과 **오늘 할 일**로 나눠 보여주는 규칙을 정의한다.
기준은 항상 **"오늘 날짜"** 와 할 일의 **`due_date`(해야 하는 날짜)** 다.

---

## 1. 분류 규칙

> 완료된 할 일(`is_done = true`)은 **어느 화면에도 표시하지 않는다.**
> 아래 규칙은 모두 **미완료(`is_done = false`)** 할 일에만 적용한다.

| 화면 | 조건 |
| --- | --- |
| **밀린 할 일** | `due_date` < 오늘  (해야 하는 날짜가 지났는데 아직 안 함) |
| **오늘 할 일** | `due_date` == 오늘 |
| (표시 안 함) | `due_date` > 오늘  (아직 미래의 할 일 — 이번 기획에서는 숨김) |

## 2. 핵심 규칙: 밀린 할 일은 사라지지 않는다

- 어제 해야 했던 할 일을 오늘까지 **완료하지 않으면**, 오늘 화면의 **"밀린 할 일"** 에 계속 보인다.
- 즉, **날짜가 지나도 완료하기 전까지는 목록에서 빠지지 않는다.**
- 완료 체크를 해야만 밀린/오늘 목록에서 사라진다.

## 3. 예시 (오늘 = 2026-06-24)

| 할 일 | due_date | is_done | 어디에 보이나 |
| --- | --- | --- | --- |
| 30분 달리기 | 2026-06-22 | false | 🔴 밀린 할 일 |
| 영어 단어 외우기 | 2026-06-24 | false | 🟢 오늘 할 일 |
| 장보기 | 2026-06-25 | false | (표시 안 함 — 미래) |
| 수학 문제집 | 2026-06-23 | true | (표시 안 함 — 완료됨) |

## 4. 화면에서 확인할 결과

- [ ] 지난 날짜의 미완료 할 일이 **밀린 할 일** 영역에 보인다.
- [ ] 오늘 날짜의 미완료 할 일이 **오늘 할 일** 영역에 보인다.
- [ ] 완료한 할 일은 두 영역 모두에서 보이지 않는다.
- [ ] (선택) 밀린 할 일은 며칠 지났는지 함께 표시하면 좋다.
