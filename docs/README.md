# 카테고리별 Todo 웹앱 — 기획 문서 (메인)

> 사용자가 **카테고리**를 만들고, 카테고리별로 **할 일(Todo)** 을 등록·관리하는 웹앱.
> 데이터는 **Supabase**에 저장하고, 최종적으로 **GitHub Pages**로 배포한다.

---

## 1. 웹앱의 목적

- 할 일을 그냥 한 줄로 쌓는 게 아니라, **카테고리(예: 공부, 운동, 집안일)** 로 나눠서 관리하고 싶다.
- 각 할 일은 **"해야 하는 날짜"** 를 가진다.
- 완료하지 않은 할 일은 **날짜가 지나도 사라지지 않고 계속 보인다**(밀린 일).
- 카테고리별로 **남은 할 일 개수**와 **평균 달성속도**(얼마나 제때/늦게 끝내는지)를 볼 수 있다.

## 2. 사용 흐름 (User Flow)

```
1) 카테고리를 만든다           →  예: "공부", "운동"
2) 카테고리를 골라 할 일을 추가  →  제목 + 해야 하는 날짜
3) 화면에서 할 일을 확인         →  [밀린 할 일] / [오늘 할 일]
4) 끝낸 할 일은 완료 체크        →  완료한 날짜가 자동 기록됨
5) 필요 없는 할 일은 삭제
6) 카테고리별 통계 확인          →  남은 개수 / 평균 달성속도
```

## 3. 화면 구성 (한눈에)

| 영역 | 내용 |
| --- | --- |
| 카테고리 목록 | 만든 카테고리들을 보여주고, 새 카테고리 추가 |
| 할 일 입력 | 제목 + 카테고리 + 해야 하는 날짜 입력 후 추가 |
| 밀린 할 일 | 해야 하는 날짜가 **오늘보다 이전**이고 아직 **미완료**인 것 |
| 오늘 할 일 | 해야 하는 날짜가 **오늘**인 것 |
| 통계 | 카테고리별 남은 개수 / 평균 달성속도 |

## 4. 화면에서 확인해야 할 결과

- [ ] 카테고리를 추가하면 목록에 바로 나타난다.
- [ ] 할 일을 추가하면 날짜에 맞는 영역(밀린/오늘)에 나타난다.
- [ ] 어제 날짜의 미완료 할 일이 **오늘도 "밀린 할 일"에 계속 보인다.**
- [ ] 완료 체크를 하면 그 할 일이 목록에서 빠지고, 완료 처리된다.
- [ ] 삭제하면 목록에서 사라진다.
- [ ] 카테고리별 남은 개수가 정확히 표시된다.
- [ ] 카테고리별 평균 달성속도가 표시된다.

## 5. 기술 스택

- **프론트엔드**: HTML / CSS / JavaScript (정적 사이트)
- **데이터베이스**: Supabase (PostgreSQL + 자동 생성 API)
- **배포**: GitHub Pages
