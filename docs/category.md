# 카테고리 기능 (Category)

> [← 메인 문서로 돌아가기](./README.md) · 관련: [데이터 구조](./data-model.md)

카테고리는 할 일을 묶는 분류 단위다. 사용자는 카테고리를 **추가**하고 **조회**할 수 있다.

---

## 1. 카테고리 추가

**무엇을 하나**
- 사용자가 카테고리 이름을 입력하고 "추가" 버튼을 누른다.
- `categories` 테이블에 새 행을 만든다.

**입력값**
- `name` (필수): 카테고리 이름

**동작 순서**
```
1) 입력칸에 이름 입력  (예: "운동")
2) [추가] 클릭
3) Supabase categories 테이블에 저장
4) 카테고리 목록을 다시 불러와 화면에 반영
```

**규칙**
- 이름이 비어 있으면 추가하지 않는다.
- (선택) 같은 이름의 카테고리는 중복으로 만들지 않는다.

## 2. 카테고리 조회

**무엇을 하나**
- 저장된 카테고리를 모두 불러와 목록으로 보여준다.
- 할 일을 추가할 때 카테고리를 고르는 선택 메뉴에도 사용한다.

**동작 순서**
```
1) Supabase categories 테이블에서 전체 조회
2) 만든 순서(created_at) 또는 이름순으로 정렬
3) 화면의 카테고리 목록 / 선택 메뉴에 표시
```

## 3. 화면에서 확인할 결과

- [ ] 카테고리를 추가하면 목록에 **즉시** 나타난다.
- [ ] 새로고침을 해도 추가한 카테고리가 그대로 남아 있다(DB 저장 확인).
- [ ] 할 일 추가 화면의 카테고리 선택 메뉴에 모든 카테고리가 보인다.
