# 카테고리 기능 (Category)

> [← 메인 문서로 돌아가기](./README.md) · 관련: [데이터 구조](./data-model.md) · [ADR 0009 수정·삭제](./adr/0009-category-edit-delete.md)

카테고리는 할 일을 묶는 분류 단위다. 사용자는 카테고리를 **추가**·**조회**·**수정**·**삭제**할 수 있다.

---

## 1. 카테고리 추가

**무엇을 하나**
- 사용자가 카테고리 이름을 입력하고 "추가" 버튼을 누른다.
- `categories` 테이블에 새 행을 만든다.

**입력값**
- `name` (필수): 카테고리 이름

**동작 순서**
```
1) 입력칸에 이름 입력  (예: "운동")
2) [추가] 클릭
3) Supabase categories 테이블에 저장
4) 카테고리 목록을 다시 불러와 화면에 반영
```

**규칙**
- 이름이 비어 있으면 추가하지 않는다.
- (선택) 같은 이름의 카테고리는 중복으로 만들지 않는다.

## 2. 카테고리 조회

**무엇을 하나**
- 저장된 카테고리를 모두 불러와 목록으로 보여준다.
- 할 일을 추가할 때 카테고리를 고르는 선택 메뉴에도 사용한다.

**동작 순서**
```
1) Supabase categories 테이블에서 전체 조회
2) 만든 순서(created_at) 또는 이름순으로 정렬
3) 화면의 카테고리 목록 / 선택 메뉴에 표시
```

## 3. 카테고리 수정

**무엇을 하나**
- 사이드바 카테고리 항목에 마우스를 올리면 나타나는 **✎(수정)** 버튼을 누른다.
  - 터치 기기에서는 버튼이 항상 보인다.
- 입력창(`prompt`)에 새 이름을 적으면 `categories` 행의 `name`을 갱신한다.

**입력값**
- `newName` (필수): 바꿀 카테고리 이름

**동작 순서**
```
1) 카테고리 항목 hover → [✎] 클릭
2) 새 이름 입력  (예: "운동" → "헬스")
3) Supabase categories 테이블의 해당 행 name 수정
4) 카테고리 목록 · 할 일 목록 · 통계를 다시 불러와 화면에 반영
```

**규칙**
- 이름이 비어 있거나 기존과 같으면 아무 것도 하지 않는다.
- **자기 자신을 제외**하고 같은 이름(대소문자/앞뒤 공백 무시)이 있으면 막는다.
  - DB UNIQUE 제약(`23505`)에 걸린 경우도 동일하게 중복 안내한다.
- 카테고리 이름만 바꾸므로, 그 카테고리를 쓰는 할 일들은 그대로 연결을 유지한다
  (할 일 목록에 보이는 카테고리 이름도 함께 갱신된다).

## 4. 카테고리 삭제

**무엇을 하나**
- 카테고리 항목의 **×(삭제)** 버튼을 누르면 확인창(`confirm`) 뒤 `categories` 행을 삭제한다.

**동작 순서**
```
1) 카테고리 항목 hover → [×] 클릭
2) 확인창 → 확인
3) Supabase categories 테이블에서 해당 행 삭제
4) 카테고리 목록 · 할 일 목록 · 통계를 다시 불러와 화면에 반영
```

**규칙**
- 삭제 전에 확인창으로 한 번 되묻는다.
- `todos.category_id`는 **`ON DELETE CASCADE`**(schema.sql)이므로, 카테고리를 삭제하면
  **그 카테고리에 속한 할 일도 DB에서 함께 삭제된다.**
  - 그래서 연결된 할 일이 있으면 확인창에 "할 일 N개도 함께 삭제됩니다"라고
    **개수를 알려 주고** 되묻는다(개수는 메모리 캐시 기준).

## 5. 화면에서 확인할 결과

- [ ] 카테고리를 추가하면 목록에 **즉시** 나타난다.
- [ ] 새로고침을 해도 추가한 카테고리가 그대로 남아 있다(DB 저장 확인).
- [ ] 할 일 추가 화면의 카테고리 선택 메뉴에 모든 카테고리가 보인다.
- [ ] 항목에 hover 하면 ✎/× 버튼이 보이고, 수정 시 목록·할 일·통계의 이름이 함께 바뀐다.
- [ ] 같은 이름으로 수정하려 하면 중복 안내가 뜬다.
- [ ] 할 일이 연결된 카테고리를 삭제하면 "할 일 N개도 함께 삭제됩니다" 경고 뒤 함께 삭제된다.
