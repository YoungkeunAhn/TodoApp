# 검증 체크리스트 (Verification)

> [← 메인 문서로 돌아가기](./README.md)

완성된 웹앱이 기획대로 동작하는지 확인하는 최종 체크리스트.
모든 항목을 직접 화면에서 눌러 보며 확인한다.

---

## 1. 카테고리

- [ ] 카테고리를 추가하면 목록에 즉시 나타난다.
- [ ] 새로고침해도 추가한 카테고리가 남아 있다. (Supabase 저장 확인)
- [ ] 할 일 추가 화면의 선택 메뉴에 모든 카테고리가 보인다.
- [ ] 카테고리 이름을 수정하면 목록·할 일·통계에 바로 반영된다.
- [ ] 카테고리를 삭제하면 그 카테고리의 할 일도 함께 삭제된다(삭제 전 개수 경고).

## 2. 할 일 추가 / 조회

- [ ] 제목·카테고리·날짜를 입력해 할 일을 추가할 수 있다.
- [ ] 입력값이 비면 추가되지 않는다.
- [ ] 추가한 할 일이 날짜에 맞는 영역(밀린/오늘)에 나타난다.
- [ ] 새로고침해도 할 일이 남아 있다. (Supabase 저장 확인)

## 3. 날짜별 화면 규칙

- [ ] 지난 날짜의 미완료 할 일이 **밀린 할 일**에 보인다.
- [ ] 오늘 날짜의 미완료 할 일이 **오늘 할 일**에 보인다.
- [ ] **완료하지 않은 할 일은 날짜가 지나도 계속 보인다.** (핵심)
- [ ] 미래 날짜의 할 일은 표시되지 않는다.

## 4. 완료 체크 / 삭제

- [ ] 완료 체크를 하면 목록에서 사라진다.
- [ ] 완료한 할 일의 `done_date`에 오늘 날짜가 기록된다.
- [ ] 삭제하면 목록에서 사라지고, 새로고침해도 다시 나타나지 않는다.

## 5. 통계

- [ ] 카테고리별 남은 할 일 개수가 정확하다.
- [ ] 할 일을 완료하면 남은 개수가 1 줄어든다.
- [ ] 카테고리별 평균 달성속도(일)가 표시된다.
- [ ] 완료 기록이 없는 카테고리는 "기록 없음"으로 표시된다.

## 6. 데이터 / 배포

- [ ] 모든 데이터가 Supabase DB에 저장된다.
- [ ] GitHub Pages 주소로 접속해 앱이 정상 동작한다.
- [ ] 배포된 사이트에서도 추가/조회/완료/삭제가 모두 동작한다.

---
